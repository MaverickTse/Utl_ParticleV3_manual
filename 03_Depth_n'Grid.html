Depth_n'Grid

Enables the emission of particle along the z-axis (depth) making the particle object truly 3D. It can also force particles to run through a grid of defined shape.

Sliders

  • z-Direction
    • The depth component for particle output direction.
  • z-Spread
    • use 180 for omni-direction emission
  • z-Gravity
    • Gravity pointing along the z-axis. Positive value cause acceleration pointing out of the screen, while negative value cause acceleration pointing towards the screen.

Setting dialog

  • GridIntvl_x(r)
    • Grid points interval along the x-axis(horizontal). As radial interval if PolarCoord is checked.
  • GridIntvl_y(theta)
    • Grid points interval along the y-axis (vertical).
    • As angular interval on the x-y plane if PolarCoord is checked.
  • GridIntvl_z(phy)
    • Grid points interval along the z-axis (depth). As partition angle on the x-z plane when PolarCoord is checked.
  • PolarCoord
    • When checked, a grid using polar coordinate with origin on object's center will be used. Otherwise, a typical Cartesian(rectangular) coordinate grid will be used.
  • GetGridInfo
    • Should be enabled only on the object that defines grid shape, and to be placed on layer N-1 if the object with ParticleRender is on layer N.
    • When using the grid feature, input non-zero values to GridIntvl_x, GridIntvl_y and GridIntvl_z and use the same numbers on the object on layer N
  • RevGrid
    • Invert transparency data of the grid.
  • Bidirectional
    • Emit particles in two directions simultaneously
  • xy_facing
    • The extra particle output direction when using Bidirectional option.
  • xy_spread
    • The extra spread angle when using Bidirectional option
  • z_facing
    • The extra output direction along depth when using Bidirectional output
  • z_spread
    • The extra spread angle along depth when using Bidirectional output
