Magnification and Transparency

Provides flexible control to particles' magnification and transparency. Parameters for magnification starts with "Mag", those for transparency starts with "Alp" (stands for alpha).

Sliders

  • StZoom
    • Initial magnification when a particle is first emitted.
    • Overrides ParticleRender@particle_ri_ver3 setting
    • Affects FineAdjustment@particle_ri_ver3
  • EdZoom
    • Final magnification when a particle reaches its survival time.
    • Overrides ParticleRender@particle_ri_ver3 setting
    • Affects FineAdjustment@particle_ri_ver3
  • StAlpha
    • Initial transparency when a particle is first emitted.
    • Overrides ParticleRender@particle_ri_ver3 setting
    • Affects FineAdjustment@particle_ri_ver3
  • EdAlpha
    • Final transparency when a particle reaches survival time.
    • Overrides ParticleRender@particle_ri_ver3 setting
    • Affects FineAdjustment@particle_ri_ver3

Check box

  • RelativeTime
    • Checked: Time zero is when a particle is emitted.
    • Unchecked: Time zero is the object's starting point.

Setting dialog

(In the following, ___ can be replaced by either Mag or Alp)

  • ___MidPtTime
    • Set the timing of mid-point(s).
    • Value in milliseconds.
    • Input format: {MP[1]Time,MP[2]Time,MP[3]Time,...,MP[n]Time}
    • The number series must be increasing. For example, {1000,2000,3000} is valid, while {2000,3000,1000} is invalid.
    • Giving only a single value will set a single mid-point.
  • ___MidValue
    • The Zoom or Transparency values at mid-points as defined by ___MidPtTime.
    • Must possess the same number count as ___MidPtTime, or gives no effect.
    • Input format: {value[1],value[2],value[3],...,value[n]}
  • ___TweenType0~2
    • 0: Instantaneous value jump
    • 1: Linear change
    • >2: Repetitive change with this value number of times. For odd number, final value is the changed one. For even value, the final value is the initial one before changing.
    • Input format: {segment[1],segment[2],segment[3],...,segment[n+1]}
    • Number count must be ONE more than ___MidPtTime
    • Leaving this field blank would assume Linear change(1)
  • ___Time_Val_Type
    • Add limited random variation to mid-points' time, value and change type.
    • Input format: {time_deviation,value_deviation,type_deviation} , while type_deviation will be effective only if >2.
  • ____ReflectMid
    • When using with Bounce@particle_ri_ver3 or Bounce2@particle_ri_ver3, use the moment of bounce as mid-point.
    • Please set this to 1, or may give error.
  • ____ScatterMid
    • When using with Style2@particle_ri_ver3, use the time of scattering as mid-point.
    • Please set this to 1 to avoid error.
  • ____LoopMid
    • Specify the mid-point after which the value will go back to initial value and loop.
    • Value is mid-point order +1
  • Sync emission
    • Synchronize variation among particles emitted at the same time when "Concurrent" option is set in ParticleRender.
